# Seedance 应用创建流程修正版

## 1. 三个对象的明确关系

### Application

Application 表示一个长期存在的应用主体。

例如：

```text
Seedance 文生视频
```

它负责：

* 应用名称
* 应用说明
* 应用图标
* 业务能力类型
* 所有者
* 可见范围
* 是否允许画布使用
* 当前默认版本
* 生命周期状态

Application 不保存具体平台参数映射，也不保存最终用户表单。

---

### ApplicationTemplate

ApplicationTemplate 表示该应用的一种底层实现方式。

例如：

```text
通过火山引擎 Seedance Pro 执行文生视频
```

或者：

```text
通过 BytePlus Seedance 2.0 执行文生视频
```

模板负责：

* 使用哪个 Provider
* 使用哪个 operation
* 固定绑定哪个 Engine，或者如何选择 Engine
* 使用哪个模型
* 哪些底层字段固定
* 哪些底层字段允许暴露
* 应用输入如何映射到底层请求
* 底层输出如何映射为应用输出
* 模板能力约束

ApplicationTemplate 可以依附于 Application，也可以先作为可复用模板创建后再绑定 Application。

第一版建议采用：

```text
Application 先创建
ApplicationTemplate 必须属于 Application
```

这样语义最简单。

---

### ApplicationVersion

ApplicationVersion 表示一个已冻结的用户契约版本。

例如：

```text
Seedance 文生视频 v1.0.0
```

它负责：

* 对用户暴露哪些输入
* 每个输入叫什么
* 输入类型
* 是否必填
* 是否可连接画布上游节点
* 默认值
* UI 控件类型
* 输出端口
* 所引用的 ApplicationTemplate
* 发布状态

ApplicationVersion 发布后不可直接修改。

---

## 2. 正确关系

```text
Application
    ├── ApplicationTemplate A
    ├── ApplicationTemplate B
    └── ApplicationVersion v1
            └── 引用 ApplicationTemplate A
```

例如：

```text
Application：Seedance 文生视频

Template A：
火山引擎
固定 Pro
最高 2K
最长 20 秒

Template B：
Seedance 官方
支持 Pro / Flash
最高 4K
最长 25 秒

Version 1.0.0：
引用 Template A
向用户暴露 prompt、resolution、duration

Version 2.0.0：
引用 Template B
向用户暴露 prompt、model、resolution、duration
```

同一个 Application 可以存在多个 Template，但每一个 ApplicationVersion 应明确引用一个具体 Template。

第一版不建议一个 ApplicationVersion 同时引用多个 Template，否则会立即引入复杂调度和多实现兼容问题。

# 3. 正确的创建顺序

## 第一步：创建 Provider 和 Engine

```text
Provider
EngineType
EngineInstance
CapabilitySnapshot
```

这一步完成平台接入和能力注册。

---

## 第二步：创建 Application

前端先让管理员填写应用基础信息。

接口：

```http
POST /api/v1/applications
```

请求：

```json
{
  "name": "Seedance 文生视频",
  "description": "根据文本提示词生成视频",
  "capability_code": "video.text_to_video",
  "visibility": "private",
  "canvas_enabled": true
}
```

返回：

```json
{
  "id": "app-seedance-text-to-video",
  "status": "draft"
}
```

此时只创建应用主体，还没有模板和版本。

---

## 第三步：进入应用配置页

前端进入：

```text
/applications/{application_id}/edit
```

页面分为：

```text
基础信息
实现模板
版本管理
发布记录
```

此时用户选择“创建实现模板”。

---

## 第四步：获取可创建模板的 Engine 和 operation

接口：

```http
GET /api/v1/applications/{application_id}/template-candidates
```

后端根据 Application 的 capability：

```text
video.text_to_video
```

筛选出所有支持该能力的 Engine。

返回：

```json
{
  "capability_code": "video.text_to_video",
  "engines": [
    {
      "engine_id": "engine-volcengine-main",
      "engine_name": "火山引擎 Seedance",
      "provider_code": "volcengine",
      "operations": [
        {
          "operation_code": "video.text_to_video",
          "models": [
            {
              "value": "seedance-pro",
              "label": "Seedance Pro"
            }
          ]
        }
      ]
    },
    {
      "engine_id": "engine-seedance-official",
      "engine_name": "Seedance 官方",
      "provider_code": "seedance-official",
      "operations": [
        {
          "operation_code": "video.text_to_video",
          "models": [
            {
              "value": "seedance-pro",
              "label": "Pro"
            },
            {
              "value": "seedance-flash",
              "label": "Flash"
            }
          ]
        }
      ]
    }
  ]
}
```

这个接口只是返回“哪些实现可以用于这个应用”。

---

## 第五步：选择 Engine 并解析模板创建表单

例如用户选择：

```text
火山引擎 Seedance
```

前端调用：

```http
POST /api/v1/applications/{application_id}/application-templates/resolve-authoring-schema
```

请求：

```json
{
  "engine_binding": {
    "mode": "fixed",
    "engine_id": "engine-volcengine-main"
  },
  "operation_code": "video.text_to_video",
  "values": {}
}
```

后端读取：

```text
Application.capability_code
Engine EffectiveCapability
ProviderOperation
CapabilityVariant
```

返回模板创建表单：

```json
{
  "fields": [
    {
      "name": "model",
      "type": "enum",
      "options": [
        {
          "value": "seedance-pro",
          "label": "Seedance Pro"
        }
      ],
      "can_fix": true,
      "can_expose": false
    },
    {
      "name": "prompt",
      "type": "string",
      "required": true,
      "can_fix": false,
      "can_expose": true
    },
    {
      "name": "resolution",
      "type": "enum",
      "options": [
        "720p",
        "1080p",
        "2k"
      ],
      "can_fix": true,
      "can_expose": true
    },
    {
      "name": "duration",
      "type": "integer",
      "minimum": 5,
      "maximum": 20,
      "can_fix": true,
      "can_expose": true
    }
  ]
}
```

因为火山引擎只支持 Pro，所以：

```text
model 不需要对终端用户暴露
model 可以在模板中固定为 seedance-pro
```

---

## 第六步：创建 ApplicationTemplate

接口：

```http
POST /api/v1/applications/{application_id}/application-templates
```

请求：

```json
{
  "name": "火山引擎 Seedance Pro 文生视频模板",
  "template_type": "saas_provider_operation",
  "engine_binding": {
    "mode": "fixed",
    "engine_id": "engine-volcengine-main"
  },
  "provider_id": "provider-volcengine",
  "operation_code": "video.text_to_video",
  "field_policies": {
    "model": {
      "mode": "fixed",
      "value": "seedance-pro"
    },
    "prompt": {
      "mode": "exposed"
    },
    "resolution": {
      "mode": "exposed",
      "option_policy": "inherit"
    },
    "duration": {
      "mode": "exposed",
      "option_policy": "inherit"
    }
  },
  "request_mapping": {
    "prompt": "prompt",
    "model": "model",
    "resolution": "resolution",
    "duration": "duration"
  },
  "output_mapping": {
    "video": "result.video_url"
  }
}
```

后端验证：

```text
Application 是否存在
Template capability 是否与 Application capability 一致
Engine 是否支持 operation
固定模型是否有效
暴露字段是否有效
模板约束是否为 Engine 能力子集
输出映射是否有效
```

返回：

```json
{
  "id": "template-volcengine-seedance-pro-t2v",
  "application_id": "app-seedance-text-to-video",
  "status": "draft"
}
```

---

## 第七步：校验模板

接口：

```http
POST /api/v1/application-templates/{template_id}/validate
```

后端执行：

```text
能力校验
请求映射校验
输出映射校验
Engine 在线校验
可选的小规模实际调用测试
```

返回：

```json
{
  "valid": true,
  "warnings": [],
  "errors": []
}
```

通过后：

```text
ApplicationTemplate.status = ready
```

---

## 第八步：基于 Template 创建 ApplicationVersion

现在才创建用户和画布看到的版本。

接口：

```http
POST /api/v1/applications/{application_id}/versions
```

请求：

```json
{
  "version": "1.0.0",
  "template_id": "template-volcengine-seedance-pro-t2v",
  "name": "Seedance 文生视频 v1",
  "inputs": [
    {
      "name": "prompt",
      "label": "提示词",
      "type": "string",
      "required": true,
      "connectable": true,
      "literal_allowed": true,
      "ui": {
        "component": "textarea"
      }
    },
    {
      "name": "resolution",
      "label": "分辨率",
      "type": "string",
      "required": true,
      "connectable": false,
      "ui": {
        "component": "select"
      }
    },
    {
      "name": "duration",
      "label": "时长",
      "type": "integer",
      "required": true,
      "connectable": false,
      "ui": {
        "component": "select",
        "unit": "秒"
      }
    }
  ],
  "outputs": [
    {
      "name": "video",
      "label": "生成视频",
      "type": "asset.video",
      "required": true
    }
  ]
}
```

这里没有 `model`，因为 Template 已将模型固定为 Pro。

返回：

```json
{
  "id": "appver-seedance-t2v-1.0.0",
  "status": "draft"
}
```

---

## 第九步：预览 ApplicationVersion 表单

接口：

```http
POST /api/v1/application-versions/{version_id}/resolve-form
```

请求：

```json
{
  "values": {}
}
```

返回：

```json
{
  "fields": {
    "prompt": {
      "visible": true,
      "required": true
    },
    "resolution": {
      "visible": true,
      "options": [
        "720p",
        "1080p",
        "2k"
      ]
    },
    "duration": {
      "visible": true,
      "minimum": 5,
      "maximum": 20
    }
  }
}
```

此时前端展示的是 ApplicationVersion 表单，而不是 Template 原始配置表单。

---

## 第十步：发布 ApplicationVersion

接口：

```http
POST /api/v1/application-versions/{version_id}/publish
```

发布时校验：

```text
Application 存在
Template 状态为 ready
输入契约与模板暴露字段一致
输出契约与模板输出映射一致
RuntimeForm 可以成功解析
至少存在一个可执行 Engine
```

发布后：

```text
ApplicationVersion.status = published
Application.default_version_id = 当前版本
Application.status = published
```

# 4. Pro 和 Flash 不拆应用时的创建流程

如果选择的是官方 Seedance Engine，并且它支持：

```text
Pro
Flash
```

则模板创建表单返回：

```json
{
  "fields": [
    {
      "name": "model",
      "type": "enum",
      "options": [
        "pro",
        "flash"
      ],
      "can_expose": true,
      "can_fix": true
    },
    {
      "name": "resolution",
      "type": "enum",
      "dynamic": true,
      "depends_on": [
        "model"
      ]
    },
    {
      "name": "duration",
      "type": "integer",
      "dynamic": true,
      "depends_on": [
        "model"
      ]
    }
  ]
}
```

用户创建 Template 时选择：

```json
{
  "field_policies": {
    "model": {
      "mode": "exposed",
      "option_policy": "allowlist",
      "allowed_values": [
        "pro",
        "flash"
      ]
    },
    "resolution": {
      "mode": "exposed",
      "option_policy": "inherit"
    },
    "duration": {
      "mode": "exposed",
      "option_policy": "inherit"
    }
  }
}
```

然后 ApplicationVersion 输入中包含：

```json
{
  "inputs": [
    {
      "name": "model",
      "type": "string",
      "ui": {
        "component": "select"
      }
    },
    {
      "name": "resolution",
      "type": "string",
      "ui": {
        "component": "select"
      }
    },
    {
      "name": "duration",
      "type": "integer",
      "ui": {
        "component": "select"
      }
    }
  ]
}
```

运行时：

```text
model = pro
→ RuntimeFormResolver 返回 Pro 的分辨率和时长

model = flash
→ RuntimeFormResolver 返回 Flash 的分辨率和时长
```

# 5. Application 和 Template 的数据库关系

## applications

```sql
CREATE TABLE applications (
    id                  UUID PRIMARY KEY,
    name                VARCHAR(256) NOT NULL,
    description         TEXT,
    capability_code     VARCHAR(128) NOT NULL,
    owner_id            UUID NOT NULL,
    visibility          VARCHAR(32) NOT NULL,
    canvas_enabled      BOOLEAN NOT NULL DEFAULT FALSE,
    status              VARCHAR(32) NOT NULL,
    default_version_id  UUID,
    created_at          TIMESTAMPTZ NOT NULL,
    updated_at          TIMESTAMPTZ NOT NULL
);
```

---

## application_templates

```sql
CREATE TABLE application_templates (
    id                      UUID PRIMARY KEY,
    application_id          UUID NOT NULL REFERENCES applications(id),
    name                    VARCHAR(256) NOT NULL,
    template_type           VARCHAR(64) NOT NULL,
    provider_id             UUID,
    operation_code          VARCHAR(128) NOT NULL,
    engine_binding_mode     VARCHAR(32) NOT NULL,
    fixed_engine_id         UUID,
    engine_selector         JSONB,
    field_policies          JSONB NOT NULL,
    request_mapping         JSONB NOT NULL,
    output_mapping          JSONB NOT NULL,
    capability_snapshot_id  UUID,
    status                  VARCHAR(32) NOT NULL,
    created_at              TIMESTAMPTZ NOT NULL,
    updated_at              TIMESTAMPTZ NOT NULL
);
```

关系：

```text
Application 1:N ApplicationTemplate
```

---

## application_versions

```sql
CREATE TABLE application_versions (
    id                  UUID PRIMARY KEY,
    application_id      UUID NOT NULL REFERENCES applications(id),
    template_id         UUID NOT NULL REFERENCES application_templates(id),
    version             VARCHAR(64) NOT NULL,
    name                VARCHAR(256) NOT NULL,
    input_contract      JSONB NOT NULL,
    output_contract     JSONB NOT NULL,
    status              VARCHAR(32) NOT NULL,
    published_at        TIMESTAMPTZ,
    created_at          TIMESTAMPTZ NOT NULL,
    updated_at          TIMESTAMPTZ NOT NULL,
    UNIQUE(application_id, version)
);
```

关系：

```text
Application 1:N ApplicationVersion
ApplicationTemplate 1:N ApplicationVersion
```

# 8. 最关键的纠正

错误链路是：

```text
直接创建 ApplicationTemplate
→ 模板请求里混入 Application 定义
→ 再创建 ApplicationVersion
```

正确链路是：

```text
创建 Application
→ 在 Application 下创建 ApplicationTemplate
→ Template 验证通过
→ 基于 Template 创建 ApplicationVersion
→ 发布 ApplicationVersion
→ 画布引用 ApplicationVersion
```

最终职责：

```text
Application
回答：这是什么应用

ApplicationTemplate
回答：它如何调用底层能力

ApplicationVersion
回答：这一版本向用户和画布暴露什么
```
