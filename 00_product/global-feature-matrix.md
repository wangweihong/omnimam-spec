# OmniMAM 全局端类型与能力标识

本文档定义领域级功能适配矩阵可以使用的端类型。领域 S1 不得自行创建新的端标识；需要新增端类型时先修改本文档。

## 端类型

| 端类型 | 使用者与定位 | 通用限制 |
| --- | --- | --- |
| Web 管理端 | 管理员和超级管理员查看只读平台能力及加载诊断，并维护系统级配置与运行环境 | 不得导入或修改 ProviderCapability；仍受不可变版本、历史引用和权限约束 |
| Web 用户端 | 普通用户在 Web 中创建、配置、运行和查看本人有权访问的业务资源 | 不展示系统凭证和底层执行实现 |
| 无限画布 | 用户通过业务节点、端口和连线组合能力的画布工作区 | 只引用稳定业务契约，不保存供应商实现细节 |
| Agent | 受用户授权后查询或调用业务能力的智能代理 | 不获得管理员配置能力，不维护平台事实副本 |
| Open API | 供受控客户端和外部集成使用的产品级接口面 | 具体协议由 S2 定义，能力不得超出对应领域 S1 |

## 支持度标记

| 标记 | 含义 |
| --- | --- |
| ✅ | 当前阶段完整支持 |
| ⚠️ | 当前阶段部分支持，领域 S1 必须说明限制 |
| ❌ | 当前阶段不支持 |
| 🚧 | 已规划但不属于当前实现范围 |

## 全局约束

- 同一业务能力在不同端可以有不同交互，但必须共享同一领域事实、权限和状态语义。
- `❌` 或 `🚧` 的能力不得由 S2 或实现擅自扩展为完整支持。
- 无限画布是端类型，不因此成为 application-platform 的领域实体；Canvas 保存态和运行态仍需由 workflow-canvas S1 定义。
