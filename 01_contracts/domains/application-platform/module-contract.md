# Application Platform Module Contract

本文档定义 `application-platform` 第一阶段 S2 模块边界。产品语义以 `00_product/domains/application-platform/product-spec.md` 为准。

## 1. 模块边界

### template

- 负责 ComfyUI 工作流模板和 SaaS API 请求模板的创建、解析、查询、更新、归档、删除预检和引用关系查询。
- 负责维护模板解析字段和模板引用计数。
- 不负责调用 SaaS API、运行 ComfyUI 工作流或管理任何运行资源。

### application

- 负责基于模板创建应用草稿、维护应用元数据、启用应用、归档应用和删除应用。
- 负责字段映射的整体保存、路径校验、展示顺序和映射有效性提示。
- 不负责后续阶段的交付、共享、通知或资源编排能力。

### access

- 负责普通用户与系统管理员在模板、应用和字段映射上的可见性与管理边界。
- 普通用户只能管理自己的模板和应用；系统管理员可以管理全部模板和应用。

## 2. 输入输出边界

| 模块 | 输入 | 输出 |
| --- | --- | --- |
| template | 模板名称、描述、类型、原始配置 | 模板记录、解析字段、引用关系 |
| application | 模板引用、应用名称、描述、字段映射 | 应用草稿、启用应用、归档应用、字段映射 |
| access | 当前用户身份、资源归属 | 可访问资源范围、权限拒绝 |

## 3. 依赖关系

- `application` 依赖 `template` 提供 active 模板和解析字段。
- `template` 依赖 `application` 提供引用关系，用于删除预检和兼容性预警。
- `access` 被 `template` 和 `application` 调用以校验资源归属和系统管理员身份。

## 4. 一致性要求

- 创建应用时必须确认模板存在且未归档。
- 启用应用前必须确认必需字段映射完整，且映射路径属于关联模板。
- 删除模板前必须确认不存在引用该模板的 draft 或 active 应用。
- 修改模板后必须重新检查引用应用的字段映射路径有效性。
- 归档应用后不得继续编辑字段映射。

## 5. 权限边界

- 普通用户只能读取和维护自己的模板、应用和字段映射。
- 系统管理员可以读取和维护全部模板、应用和字段映射。
- 第一阶段不定义公共应用，因此不存在匿名访问或跨用户公共可见性。

## 6. 事件边界

- `template` 生产 `app_template_changed`。
- `application` 生产 `application_changed` 和 `field_mapping_changed`。
- 事件用于模块内一致性维护，不表达后续阶段交付、共享、通知或资源编排语义。

## 7. 非目标范围

第一阶段模块契约不包含计划存档中的后续能力。归档内容不作为当前 S2 契约来源。
