# S2 实现契约事实源规则

S2 是实现契约事实源，是 S1 的结构化表达。

S2 用于指导实现和自动化校验。

---

## 1. S2 定位

S2 负责定义：

```text
API
请求响应结构
设计态 SQL schema
错误码
权限码
事件契约
模块契约
```
S2 不能凭空创造 S1 中未定义的核心业务能力。

## 2. 文件位置
```
01_contracts/domains/<domain_id>/
├── openapi.yaml
├── schema.sql
├── errors.yaml
├── permissions.yaml
├── events.yaml
└── module-contract.md
```
## 3. S1 追溯性要求

所有 S2 契约必须能追溯到 S1 元素。

必须显式引用相关 US/BR 的内容：
```
OpenAPI operation
错误码
权限码
事件
模块契约
```
schema.sql 至少在表级标注来源；关键字段可补充字段级注释。

推荐引用字段：
```
x-s1-refs:
  user_stories:
    - US-AICHAT-001
  business_rules:
    - BR-AICHAT-002
```
或使用等效字段：
```
related_user_stories:
  - US-AICHAT-001
related_rules:
  - BR-AICHAT-002
```
要求：

引用编号必须真实存在
引用编号必须属于当前或明确依赖的 domain
不得引用不存在、废弃或语义无关的编号

## 4. openapi.yaml 规则

openapi.yaml 覆盖所有需要服务支持的 S1 用户故事。

必须定义：
```
接口路径
HTTP 方法
请求参数
请求体
响应结构
错误响应
状态码
鉴权要求
分页规则
排序规则
过滤规则
x-s1-refs
```
禁止：
```
先实现后反推 OpenAPI
OpenAPI 与 S1 用户故事长期不一致
OpenAPI 中出现 S1 未定义的核心业务能力
```
## 5. schema.sql 规则

schema.sql 是设计态表结构，不是实际 migration。

必须定义：
```
表
字段
字段类型
主键
唯一约束
外键关系
索引建议
默认值
软删除策略
时间字段
JSON 字段约束
状态字段取值
S1 来源注释
```
建议用 SQL 注释标注来源：
```
-- s1_refs: US-AICHAT-001, BR-AICHAT-002
CREATE TABLE ai_assistants (
  id UUID PRIMARY KEY
);
```
## 6. errors.yaml 规则

错误码必须来自 S1 的业务规则或异常场景。

推荐结构：
```
ERROR_CODE:
  message:
  http_status:
  module:
  meaning:
  frontend_behavior:
  retryable:
  related_rules:
    - BR-XXX-001
  related_user_stories:
    - US-XXX-001
```
要求：
```
唯一
稳定
语义清晰
HTTP 状态码明确
前端展示行为明确
必须关联 S1
```
## 7. permissions.yaml 规则

权限码必须来自 S1 的角色、可见性和操作规则。

推荐结构：
```
PERMISSION_CODE:
  module:
  resource:
  action:
  description:
  frontend_usage:
  backend_enforcement:
  related_rules:
    - BR-XXX-001
  related_user_stories:
    - US-XXX-001
```
要求：
```
唯一
稳定
前后端共用
前端可用于展示控制
后端必须用于真实校验
必须关联 S1
```
## 8. events.yaml 规则

事件定义必须来自 S1 的异步流程、任务状态、通知规则或跨模块协作需求。

推荐结构：
```
event_name:
  producer:
  consumers:
    - consumer_name
  trigger_condition:
  payload:
  idempotency_key:
  retry_policy:
  failure_policy:
  related_rules:
    - BR-XXX-001
  related_user_stories:
    - US-XXX-001
```
要求：
```
事件名唯一
payload 明确
生产者明确
消费者明确
触发时机明确
幂等规则明确
失败处理明确
必须关联 S1
```
## 9. module-contract.md 规则

module-contract.md 用于定义模块职责和边界。

应包含：
```
模块职责
模块不负责什么
输入
输出
依赖模块
被依赖模块
跨模块调用规则
数据归属
权限边界
事件边界
相关 S1 引用
```
模块契约用于防止模块边界混乱和跨模块穿透。

## 10. S2 与架构参考

架构参考文件：
```
02_architecture/global-architecture.md
02_architecture/domains/<domain_id>.md
```
用于说明：
```
全局模块关系
领域模块关系
核心时序
运行时边界
外部 provider 边界
跨模块依赖
```
架构参考不替代 S1/S2。

冲突时：
```
产品语义以 S1 为准。
实现契约以 S2 为准。
架构参考需同步修正。
```
## 11. 最终规则
S2 是实现契约事实源。
S2 必须从 S1 推导。
S2 必须可追溯到 S1。
S2 不得凭空新增 S1 未定义的核心业务能力。
